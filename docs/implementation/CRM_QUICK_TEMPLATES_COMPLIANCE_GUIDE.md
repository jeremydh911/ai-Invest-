A Quick Template is a pre-configured role definition that includes:

1. **Job Description** (400-600 lines)

## Compliance Frameworks

### 1. DLP (Data Loss Prevention)

#### Purpose
Prevents sensitive data from being shared externally or in unsecured communications.

#### Data Patterns Detected

**Financial Data**
- Bank account numbers (8-17 digits)
- Bank routing numbers (9 digits)
- Credit card numbers (13-19 digits)
- Routing numbers and ACH information

**Personal Identification**
- Social Security Numbers (XXX-XX-XXXX)
- Driver license numbers
- Passport numbers

**Healthcare Data**
- Patient ID numbers
- Medical record numbers
- Health conditions and treatments
- Medication information

**Real Estate Data**
- Property pricing (when external)
- MLS numbers and listing data
- Agent contact information

#### Enforcement Levels

| Level | Action | Block | Warn | Allow |
|-------|--------|-------|------|-------|
| **Strict** | Automatic blocking | SSN, Account#, Credit Card | Routing#, Patient ID | None |
| **Moderate** | Warning & block | Patient ID | Pricing, MLS# | Email, Internal |
| **Low** | Warning only | None | Sensitive data | Most content |

#### Implementation Examples

```javascript
// Block SSN in any context
SSN_PATTERN: /\b\d{3}-\d{2}-\d{4}\b|(?<!\d)\d{9}(?!\d)/
Context: BLOCK in email, web, print, internal

// Warn on property pricing external
PROPERTY_PRICE: /sold for|sale price|list price.*\$\d+/i
Context: BLOCK in web, WARN in email

// Allow MLS data with restrictions
MLS_PATTERN: /\b[Mm][Ll][Ss]\s+#?\d+\b/
Context: ALLOW internally, BLOCK in public web
```

### 2. Banking Compliance (GLBA / SOX)

#### GLBA (Gramm-Leach-Bliley Act)

**Three Core Pillars**

1. **Financial Privacy Rule**
   - Collect only necessary financial information
   - Provide privacy notices to customers
   - Limit collection to stated purpose

2. **Safeguards Rule**
   - Develop security program
   - Designate security officer
   - Assess risks and implement controls
   - Monitor systems and test effectiveness
   - Oversee third-party vendors

3. **Disposal Rule**
   - Safely dispose of customer financial information
   - Protect against unauthorized access
   - Document disposal procedures

#### SOX (Sarbanes-Oxley Act)

**Section 302: Certification**
- CEO and CFO must certify financial statements
- Personal responsibility for accuracy
- Criminal penalties for false certification

**Section 404: Internal Control Assessment**
- Establish internal control framework
- Document all critical processes
- Test controls for effectiveness
- Report on control effectiveness

**Section 906: Criminal Penalties**
- Criminal penalties for false certification
- Up to 20 years imprisonment
- Fines up to $5 million

#### Key Controls Required

| Control | Requirement | Enforced By |
|---------|-------------|------------|
| **Access Control** | Limit financial data access | Role-based permissions |
| **Encryption** | Encrypt data in transit & at rest | AES-256 minimum |
| **Audit Trail** | Log all financial transactions | 7-year retention |
| **Approval Authority** | Require approval for transactions | Workflow enforcement |
| **Segregation of Duties** | Separate authorization, execution, custody | User role restrictions |

### 3. HIPAA (Health Insurance Portability and Accountability Act)

#### Scope
Applies when handling Protected Health Information (PHI) or health benefits data.

#### Key Definitions

**Protected Health Information (PHI)**
- Any health information that can identify an individual
- Includes: names, medical record numbers, health conditions, medications, treatment plans
- Includes: health plan information and payment information for healthcare

**Personally Identifiable Information (PII) in Healthcare Context**
- Name, address, phone number, email
- Social Security number
- Medical record number
- Health plan beneficiary number
- Dates of birth and dates of service

#### Three Rules

**1. Privacy Rule**
- Minimum necessary principle: Only use minimum required PHI
- Use limitation: PHI only used for stated purpose
- Patient rights: Access, amend, accounting of disclosures
- De-identification: Can use if identifiers removed (18 specific items)

**2. Security Rule**
- Administrative safeguards: Policies, training, access controls
- Physical safeguards: Facility access, workstation security
- Technical safeguards: Access controls, encryption, audit logs

**3. Breach Notification Rule**
- Must notify individuals if PHI is breached (60 days or sooner)
- Must notify HHS for all breaches
- Must notify media if 500+ residents affected
- Must document breach information

#### Penalties for Non-Compliance

| Violation Type | Penalty | Prison |
|---|---|---|
| Unknowing violation | $100-$50,000 per violation | Up to 1 year |
| Intentional violation | $1,000-$50,000 per violation | Up to 10 years |
| False pretenses | $10,000-$50,000 per violation | Up to 10 years |

### 4. MLS & Fair Housing

#### MLS Rules

**Data Integrity**
- Accurate property information
- Timely status updates
- Proper price modifications
- Correct expiration dates

**Confidentiality**
- Don't share pending sale data without authorization
- Protect agent contact information
- Respect seller privacy

**Professional Conduct**
- No misrepresentation
- No false advertising
- Professional treatment of other agents
- Maintain confidentiality

#### Fair Housing Compliance

**Protected Classes (7)**
1. Race or color
2. Religion
3. Sex
4. National origin
5. Familial status
6. Disability
7. Sexual orientation (some jurisdictions)

**Prohibited Actions**
- Steering: Guiding people to certain properties/neighborhoods
- Redlining: Refusing service to certain areas
- False advertising: Misrepresenting properties or terms
- Discriminatory terms: Using discriminatory language
- Different treatment: Treating people differently
- Harassment: Any harassment based on protected class

**Required Practices**
- Annual fair housing training
- Non-discriminatory marketing
- Equal access to all properties
- Document equal treatment
- No restricted communications

## DLP Policy Enforcement

### How DLP Works

1. **Content Inspection**
   - User attempts to send email/share data
   - System scans for sensitive patterns
   - Matches against DLP policy

2. **Pattern Matching**
   - SSN: `\b\d{3}-\d{2}-\d{4}\b`
   - Account: `\b\d{8,17}\b`
   - Credit Card: `\b(?:\d[ -]*?){13,19}\b`

3. **Action**
   - **BLOCK**: Content not sent, user notified, logged
   - **WARN**: User warned, can proceed with acknowledgment
   - **ALLOW**: Content sent without restriction

4. **Logging**
   - All violations logged to audit trail
   - Admin alerted for severe violations
   - Reports generated for compliance

### DLP Examples

#### Example 1: SSN in Email (BLOCKED)
```
User writes: "Employee John Smith (123-45-6789) needs benefits review"

System action: EMAIL BLOCKED
```

## Best Practices

### For Administrators

1. **Template Selection**
   - Review job description carefully
   - Verify compliance requirements match organizational needs
   - Ensure setup requirements are realistic
   - Plan training timeline

2. **Setup Management**
   - Assign a setup administrator
   - Track completion percentage
   - Document all training and certifications
   - Collect signed agreements

3. **Compliance Verification**
   - Verify all certifications before activation
   - Document all training completion
   - Maintain signed agreements
   - Conduct background checks as required

4. **Ongoing Monitoring**
   - Review audit logs regularly
   - Monitor DLP violations
   - Track compliance renewals
   - Update training as needed

### For Users

1. **Setup Completion**
   - Complete all required training
   - Obtain necessary certifications
   - Review and sign agreements
   - Set up security devices (MFA, security keys)

2. **DLP Compliance**
   - Never send SSNs via email
   - Use encrypted channels for sensitive data
   - Follow confidentiality guidelines
   - Report suspicious requests

3. **Ongoing Compliance**
   - Renew certifications on schedule
   - Attend refresher training
   - Report compliance questions
   - Maintain audit trail awareness

### For Security/Compliance Teams

1. **Policy Enforcement**
   - Monitor DLP violations
   - Review access logs
   - Track compliance metrics
   - Generate compliance reports

2. **Incident Response**
   - Investigate DLP violations
   - Document findings
   - Take corrective actions
   - Update policies as needed

3. **Continuous Improvement**
   - Review policy effectiveness
   - Update policies as regulations change
   - Provide training on new requirements
   - Benchmark against industry standards
